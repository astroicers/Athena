Remote Desktop Protocol (RDP) on port 3389 is a common lateral movement vector because it provides a full interactive GUI session and is widely enabled in enterprise environments, often with weak credential policies. Attackers use `xfreerdp` from Linux to connect using stolen credentials or NTLM hashes (Restricted Admin mode), while on Windows `mstsc.exe` provides the native client. Session hijacking via `tscon` allows an attacker with SYSTEM privileges to take over disconnected RDP sessions without knowing the original user's password, and sticky keys backdoors (replacing `sethc.exe` with `cmd.exe`) can provide pre-authentication SYSTEM shells on locked workstations.
