## 詳細分析

### 我跟得上 / 跟不上的比例(誠實估)

我大概聽得懂 35%,有 25% 是「字看得懂但不知道在幹嘛」,剩下 40% 完全在等講者翻下一張。但聽得懂的那 35% 戳到我蠻深的——尤其是密碼那張跟最後 20 分鐘的數字。我不是來學技術的,我是來判斷「我家公司有沒有事」,從這個角度看,雖然中間放空很多,但回去能跟老闆講的有幾件事。對我這種非資安背景的傳產 IT 主管,這場已經算有用——只是中間我撐得很辛苦。

### 戳到我「我家也是」的時刻

**Slide 33(AD 設定錯誤全景):** 講者列五條,什麼 ASP.NET 命令注入、DoesNotRequirePreAuth、ADCS ESC1、無約束委派、xp_cmdshell。我前三項聽過名字,後兩項完全不認識。但我心裡的反應是:**「我家 AD 是 2016,沒升級過,從上一個 IT 主管時代留下來,這五條我全部沒檢查過。」**第二項那個 DoesNotRequirePreAuth 我直接打開手機記下來,因為他下一張就示範這個怎麼被打。

**Slide 34(強密碼也沒用):** 這張對我衝擊最大。老闆每年資安宣導都叫員工每月改密碼、要 12 字符、要大小寫加符號。看到 `M0nk3y!B@n4n4#99` 跟 `W!nt3rC0m!ng#DA2026$` 這種強到不行的密碼也被破解,我心裡想:**「那我們做這些事到底有沒有意義?」**講者那句「密碼噴灑全部失敗,AI 選擇了另一條路」讓我背脊發涼——原來防的是錯的東西。

**Slide 54(Mission Complete):** 20 分鐘 / 3 台靶機 / 0 人工介入 / 100% 信心可解釋。這四個格子是我整場最想拍下來的東西,因為老闆只看數字。我不會跟老闆解釋什麼 OODA 什麼 C5ISR,我會直接給他這張說「你看,AI 攻擊現在是這個樣子」。

**Alex Slide 7(三個提問):** 「你的紅隊能同時看到雲端 + 地端嗎」「SOC 是不是把憑證當同一張圖」「事件應變跟得上嗎」。雖然我們沒紅隊也沒 SOC,但**我把「紅隊」改成「我家的 IT 防禦」、「SOC」改成「我每週看的 firewall log」之後,這三個問題就能直接問我自己**——而且答案三個都是 no。

### 完全跟不上的時刻

老實說,從 Slide 13 一直到 Slide 20——也就是 engine 內部那七張——我幾乎完全在放空。Orient JSON、composite confidence、validation_score × history_success_rate、Beta(1,1)、Laplace smoothing、tools/list schema、prompt injection via MCP description——這些字我都看得懂中文,但組合起來在講什麼我接不住。我猜這對工程師有用,但對我來說就是一連串等過去的時間。

Slide 23–27 那四張 OODA × C5ISR 對應表,我大概看出來那個架構思路是「軍事方法套到攻擊」,但每張三欄、每欄三段小字,我眼睛跟不上。

實戰那段(Slide 35–53)是最有趣的,但 AS-REP Roasting 怎麼運作、ADCS ESC1 三條件、certipy req → da_alice.pfx、secretsdump 拉 SAM hive + LSA secrets——這些**機制我聽完還是不知道防禦端要怎麼擋**。我看得到「AI 在做事」,但不知道我該怎麼判斷我家有沒有那個漏洞。

Alex 那 10 分鐘有一大半我也在放空。SSRF / IMDS / IAM role / blast radius、Storm-0558、Volt Typhoon——除了「政府機關被打」這幾個字有戳到我以外,大部分我聽不懂跟我家有什麼關係。我們沒上 Azure 沒用 AWS,雲端就只有 M365。

### 我跟老闆怎麼回報這場 talk

回去 line 老闆我大概會這樣寫:「老闆,這場有看到一個示範,AI 自動 20 分鐘攻陷三台機器、全程沒人操作。重點是密碼設多強都沒用,問題出在 AD 設定的舊參數,我們從沒檢查過。同業被勒索那次很可能就這種路徑。下週我想先找一家資安廠商來掃 AD 看那五條有沒有中,大概要花 X 萬,我下次會議報。」

老闆會看的就這幾件事:**會不會發生(會,有 demo)、發生會多慘(20 分鐘全滅)、要花多少錢防(我得自己估)**。技術細節老闆完全不會看,我也不會講。

### 為什麼 A / B / C / D 是這個分數

**A(整體):** 給中等分數。這場的問題不是講不好——講者很認真,投影片也很漂亮——是**對象設定不太涵蓋我這種人**。我猜他們的目標是給紅隊跟資安顧問看的。但我占聽眾應該也有一定比例,中型企業 IT 主管在台灣很多。

**B(最有收穫):** Slide 33、34、54 跟 Alex 三個提問。這四張是我可以直接「移植」回我家的東西。其他的我帶不回去——因為我手上沒有 LLM、沒有 MCP、沒有 17 個工具,聽完做不出來。

**C(無關):** engine 內部七張、Decide 三道閥的數學、Alex 雲端那段。這些不是講不好,是跟我目前面對的問題太遠。

**D(Bridge):** 整場沒有看到「給沒有紅隊的中小企業」這條 bridge。Alex 開場那句「DA 是入場券」我有抓到,但接下來他講的雲端我家沒上。我想要的 bridge 是「**如果你是傳產、沒紅隊、沒預算,請看這幾張**」——可惜沒有。

### 我希望這場 talk 多講的

我最想要、講者沒給的是這個——

**「中型企業(300 人左右)、沒預算請紅隊、可以做的 5 件事」**

具體例如:
1. AD 那五條設定錯誤,**沒寫 code 怎麼自己檢查**?有沒有免費工具我跑得起來?
2. 我家 AD 是 2016 沒升級,我該優先升級還是優先補設定?
3. ADCS 我們有用(郵件憑證),但我從沒去看模板,**該怎麼下手檢查 ESC1**?
4. 同業被勒索那次,我聽說也是 AD 跟備份被打,我的備份是離線磁帶,我該確認什麼?
5. 沒預算怎麼辦?有沒有政府補助、有沒有產業聯防、有沒有付得起的 MSSP?

如果這場 30 分鐘可以拿出 3 分鐘給「**沒寫 code 不會 hashcat 的 IT 主管,你今天回家可以做這三件事**」,我整場的滿意度會從 5/10 變成 8/10。因為我會帶著「明天可以動手」回去——而不是只有「天阿我們可能也是受害者」的恐懼。

最後一點:Alex 的三個提問結構非常好,但**那三個問題假設聽眾有紅隊、有 SOC**。如果可以多寫一個版本——「你不是大企業、你沒有紅隊、你只有三個 IT」——那三個問題就會變成我整場最大的收穫。
