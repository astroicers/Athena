SSH agent forwarding allows a user to authenticate to intermediate hosts using keys stored on their local machine, but if an attacker achieves root access on a jump host, they can hijack agent sockets belonging to other users' forwarded SSH agents. The attack involves finding SSH agent socket files under `/tmp` (typically `ssh-XXXXXXXXXX/agent.PID`), setting the `SSH_AUTH_SOCK` environment variable to point to a victim's socket, and then using `ssh-add` and `ssh` as if the keys were their own. This technique requires only filesystem access to the socket file and does not expose the private key material itself, making it stealthy and difficult to detect without agent-level audit logging.
