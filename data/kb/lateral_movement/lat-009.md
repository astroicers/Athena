Password spraying attempts a small number of commonly used passwords (e.g., seasonal passwords like `Winter2024!`, company name variants) against a large list of valid usernames, circumventing account lockout policies that trigger after multiple failed attempts for a single account. Unlike brute-forcing a single account, spraying distributes attempts across many accounts, making it significantly harder to detect with standard lockout-based alerting. Captured hashes from previous steps can be cracked offline with Hashcat using wordlists like `rockyou.txt` combined with rule files to generate mangled variants, while tools like Kerbrute and CrackMapExec automate spraying against Active Directory environments.
