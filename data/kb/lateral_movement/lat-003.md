PSExec achieves remote command execution by authenticating over SMB (port 445), uploading a service binary to the ADMIN$ share, creating a Windows service on the remote host, and executing it — all in a single workflow. Sysinternals PsExec requires valid credentials and spawns an interactive session, while Impacket's psexec.py supports both password and hash-based authentication, making it suitable for pass-the-hash scenarios. The technique generates distinctive artifacts including service creation events (Event ID 7045) and PSEXESVC service entries in the Windows registry, making it detectable by mature EDR and SIEM solutions.
