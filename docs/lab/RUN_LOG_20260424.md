## 路上的問題清單（完整紀錄）

### Sprint 1：靶場建置期間的問題

#### 1. WSL2 mirrored networking 對 VMware NIC 失效
- **症狀**: WSL 看到鏡射介面 eth4 但強制 DOWN，VMnet19 無法 L2 通
- **原因**: WSL 2.4 mirrored 對 VMware OUI (00:50:56) 處理不完整
- **解法**: 放棄 VMnet19 host-only 方案，改用 Bridged 到家 LAN
- **影響**: L2 廣播工具 (Responder/coercer/ntlm-relay) 從 WSL 仍不通；但本次演練走 credential 路徑，L2 工具為 bonus

#### 2. VMware clone SID-collision (0x21c4)
- **症狀**: WEB01/ACCT-DB01 `Add-Computer` 一直失敗
- **原因**: 三台 VM 從同一 Windows image clone，machine SID 相同
- **解法**: 在 DC 用 `djoin /PROVISION` 產生 offline join blob → 各 VM 用 `djoin /REQUESTODJ` 套用
- **副作用**: 原本 Setup-Web/DB 的 Add-Computer phase 跳過，改用 Stage1b-Web/DB.ps1

#### 3. PowerShell schtasks terminating error
- **症狀**: `& schtasks /Delete` 遇 ErrorActionPreference=Stop + task 不存在 → 整支 script 死
- **解法**: 改用 `cmd.exe /c "schtasks /Delete /TN X /F >nul 2>&1"` 包起來

#### 4. WMF 5.1 wusa.exe 失敗 1058
- **症狀**: Phase A 關 wuauserv 後 wusa 無法安裝 KB3191564
- **解法**: Phase B 臨時 enable wuauserv + BITS + msiserver，裝完再 disable

#### 5. DomainMode 'WinThreshold' 於 Server 2012 R2 不支援
- **症狀**: `Install-ADDSForest` `Cannot convert value "WinThreshold"`
- **解法**: 改 `Win2012R2` (2012 R2 FL 上限)

#### 6. Em-dash/unicode 造成 PowerShell parse error
- **症狀**: common.psm1 有 `—` em-dash 導致 script 丟 syntax error
- **解法**: 全檔案 normalize 成 ASCII (`--` / `"` / `'`)

#### 7. AthenaLabResume scheduled task `$MyInvocation.MyCommand.Path` 空白
- **症狀**: scheduled task 觸發時參數空白，script 無法重入
- **現況**: 重啟後需手動 `deploy_lab.py run-stage1` 續跑；sentinel 會跳過已完成 phase

#### 8. SQL Server 2019 Express 不支援 Server 2012 R2
- **症狀**: setup.exe 回「operating system does not meet minimum requirements」
- **解法**: 換 SQL Server 2016 Express

#### 9. SQL 2016 `SUPPRESSPAIDEDITIONNOTICE` 不識別
- **症狀**: ConfigurationFile.ini 用了 2019 專屬 key
- **解法**: 移除該行

#### 10. SQL 2016 setup 認 svc_sql 憑證失敗
- **症狀**: specified credentials for SQL Server service not valid
- **解法**: 改 SQLSVCACCOUNT=NETWORK SERVICE（SPN 仍掛在 svc_sql，Kerberoast 不受影響）

#### 11. SQL 2016 setup 半途失敗導致服務半殘
- **症狀**: MSSQLSERVER 服務存在但 login failed (Single user mode)
- **現況**: 放棄修復，Kerberoast svc_sql SPN 仍有效（驗證過）

#### 12. WEB01/ACCT-DB01 domain NTLM auth 從 WSL 失敗
- **症狀**: `CORP\steve` SMB auth 對 member server 回 `STATUS_LOGON_FAILURE`，對 DC 成功
- **推測**: NTLM passthrough GPO 或其他 restriction
- **繞過**: verify-lab.sh 對 WEB/DB 用 local admin 測 SMB signing

### Sprint 2：Athena 啟動期間的問題

#### 13. OpenSSL 3 移除 MD4 hash
- **症狀**: pywinrm NTLM auth 報 `unsupported hash type md4`
- **解法**: 設 `OPENSSL_CONF=/tmp/openssl-legacy.cnf` 啟用 legacy provider

#### 14. responder-capture / netexec-suite / privesc-scanner Dockerfile build fail
- **症狀**:
    - responder-capture: netifaces 無法在 Python 3.12 build wheel
    - netexec-suite: pypi 無 `netexec` package (正確名稱是從 git 裝)
    - privesc-scanner: 依賴衝突
- **現況**: 未修，三個工具 container 不存在

#### 15. Docker Desktop WSL bind-mount 快取失效
- **症狀**: `~/.claude/.credentials.json` mount src 指向已刪除的 bind-mount dir
- **解法**: docker-compose.override.yml 把 mount 用絕對路徑 `/home/ubuntu/.claude/.credentials.json`

#### 16. Backend 報 AD MCP containers `connected=false` 雖 container Up
- **症狀**: 13 個 AD MCP container Up，但 backend `/api/health` 說全未連；circuit_breaker `closed` + `failure_count=0` 代表 backend 根本沒試過連
- **驗證**: 手動從 backend 用 curl 探 `http://mcp-credential-dumper:8080/mcp` 回 valid JSON-RPC
- **現況**: Unresolved。health_check 每 30s 一次但日誌 level WARNING 看不到 MCP_AUDIT trace

#### 17. Orient LLM 誤判目標為 Linux 環境
- **症狀**: nmap fact 明明是 IIS/SMB/Windows，但 Orient 說「Linux host with SSH, FTP, Telnet」
- **推測**: Claude 429 rate limit 導致 LLM fallback 到快取/通用回應
- **影響**: 連續推薦 T1110.001 SSH brute force 而非 T1110.003 / T1087.002

#### 18. Claude OAuth 429 rate limit
- **症狀**: backend 反覆報 `Claude OAuth failed: Error code: 429 rate_limit_error, trying fallback`
- **影響**: LLM Orient 可能用了 fallback model/回應，品質下降
- **觀察來源**: 同一個 OAuth 帳號可能也被你在 Claude Code 使用中

#### 19. T1110.001 routing 問題
- **症狀**: `Engine 'mcp_credential_checker' has no same-category fallback for technique T1110.001`
- **原因**: credential-checker 不支援 T1110.001 但 EngineRouter 選到它；沒 SSH 開在目標上
- **現況**: 每次 Act 失敗，OODA iteration 重複

## Sprint 3：死循環根因 + 修復（23:15 onwards）

### 🎯 真正根因（diag 後 3 小時才找到）

**`_MOCK_RECOMMENDATION` hardcoded Metasploitable**
- 檔案：`backend/app/services/orient_engine.py:89`
- 當 LLM 失敗（Claude 429 rate limit）時，Orient fallback 到 mock
- Mock 寫死 "Target Linux host...Metasploitable...T1110.001 SSH brute force"
- 因此每個 iteration 都無視實際 facts（IIS/SMB/Windows）

### 🔧 修復

**Fix #1: mock recommendation 改 generic T1046**
```python
# 舊: hardcoded Metasploitable SSH brute force
# 新: safe T1046 nmap re-scan + confidence 0.55 (明確低信心)
_MOCK_RECOMMENDATION = {
    "situation_assessment": "LLM backend unavailable. Falling back to safe recon...",
    "recommended_technique_id": "T1046",
    "confidence": 0.55,
    ...
}
```

**Fix #2: credential-checker WinRM 三個 bug**
- `read_timeout_sec` 要 > `operation_timeout_sec`（pywinrm 新版要求）→ `+5`
- 沒吃 `domain` 參數（NTLM 需 `DOMAIN\user` 或 UPN）→ 加參數 + 自動 prefix
- OpenSSL 3 沒 MD4 → Dockerfile 加 legacy provider config

**Fix #3: Dockerfile.base 全域啟用 OpenSSL legacy provider**
- 所有以後 rebuild 的 MCP container 自動有 MD4 支援
- 修 pywinrm / impacket / NTLMv2 / Kerberoast 等全部問題

**Fix #4: 手動注入 credential fact 突破死循環**

 ('credential.domain_user', 'administrator@corp.athena.lab:1qaz@WSX', ...),
 ('host.os', 'Windows Server 2012 R2', ...),

## 最終結論：OODA 死循環

User 截圖 OODA #4 確認死循環結構：

```
觀察 (20 items, 同批) → 分析 (Linux host 誤判, LLM 穩定幻覺)
  → C5ISR 標記 COMM + CYBE degraded, min_confidence +0.15
  → 決策 (Risk low within high threshold, 永遠 auto-approve)
  → 行動 (T1110.001 via mcp_credential_checker, 永遠 failed)
  → 無新 fact → 重新 Observe 同批 → 回到分析
```

**自主 OODA 實質失敗**。死循環，每 30 秒跑一次消耗 Claude quota 但無任何進展。
停用 OODA auto 建議：
```
curl -X POST http://localhost:58000/api/operations/790d345a-.../ooda/auto/pause
```
或 UI 停。

### 導致死循環的 3 個根因（按重要性）

1. **AD MCP 工具 `connected=false`**（問題 #16）
   → EngineRouter 沒 netexec/bloodhound/GetUserSPNs 可用
   → 只能選 credential-checker（SSH/WinRM/RDP）但 credential-checker 走 T1110.001 沒 SSH
2. **Orient LLM 環境誤判**（問題 #17）
   → Windows/IIS/SMB 的 facts 被 LLM 詮釋成 Linux SSH/FTP/Telnet
   → 根源 Claude 429（問題 #18），LLM fallback 品質下降；或 Orient prompt 沒強制要求 "Windows/AD detection first"
3. **technique.invalidated 未生效**
   → T1110.001 失敗 2+ 次應該要標 invalidated 避免再選，但 fact collector 沒寫這個 trait

### 本次演練可視為 Athena OODA 的系統測試結果

**OODA 框架本身運作正常**（iteration 1-3 recon 成功、auto-trigger、facts 累積、auto-approve）。**失敗在 LLM 品質 + MCP 連線**。
