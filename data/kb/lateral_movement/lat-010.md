SSH tunneling enables attackers to pivot through compromised hosts by routing traffic to network segments that are not directly accessible from the attacker's machine, using the SSH protocol as an encrypted transport layer. Local port forwarding (`-L`) proxies traffic from the attacker's machine through the jump host to a target; remote port forwarding (`-R`) exposes a service on the attacker's machine to the internal network; and dynamic forwarding (`-D`) creates a SOCKS5 proxy for routing arbitrary TCP traffic through tools like `proxychains`. When SSH access is unavailable, similar pivoting can be achieved with tools like `chisel`, `ligolo-ng`, or Metasploit's `portfwd` command.
